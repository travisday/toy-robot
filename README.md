# Toy Robot Simulator

## Tech Stack
```text
Backend - NestJS
Database - SQLite
Frontend - React Rotuer v7
```

## Project structure

### Frontend
```text
app/
    components/
        ui/                 # ShadCn auto generated components
        table.tsx           # component to rend the table grid
    lib/
        api.ts              # utlity functions to call api
        types.ts            # utility to decalre types used
        utils.ts            # auto generated by shadcn
    routes/
        home.tsx            # main route 
    simulator/
        logic.tsx           # functions to handle robot logic
        simulator.test.tsx  # test script for simulator
        simulator.tsx       # main component for robot state and functions
    routes.ts               # entry point
```
### Backend
```text
src/
    positions/
        dto/
            create-position.dto.ts      # postions dto schema
            update-position.dto.ts      # postions dto schema
        entities/
            position.entity.ts          # positions schema
        positions.service.ts            # service functions for positions resource
        positions.module.ts             # module for positions resource
        positions.contoller.ts          # controller for positions resource
    main.ts                             # Nest boilerplate
    app.service.ts                      # Nest boilerplate
    app.module.ts                       # Nest boilerplate
    app.contoller.ts                    # Nest boilerplate
```

## How to run

### Backend

1. cd into `backend`
2. run `npm install`
3. run `npm run start`
4. api will be running on `localhost:3000`

### Frontend

1. cd in `frontend`
2. run `npm install`
3. run `npm run dev`
4. frontend will be running on `localhost:5173`

## Testing

1. navigate to `localhost:5173` in your browser
2. click a postion on the table to PLACE the robot
3. use the buttons or arrow keys to turn and MOVE the robot around the table
4. clicking REPORT will output the current position and diretion the robot is facing
5. refresh the page and notice the robots position is persisted
6. in a separate browser window navigate to `localhost:3000/positions` and you can see all the saved positions of the robot in the db

## Assumptions
1. Since the project doc mentioned not being pixel perfect I made the assumption to design the frontend how I saw fit. I used readily availbale ShadCN components to be mindful of my time spent.

2. Assumed since this is just a 4 hour assesment I did not need to add any api or database security

3. Since it wasn't specifcally mentioned in the porject doc I made the assumption I could choose how the arrow keys behaved. The left and right arrow keys change the direction of the robot and the up arrow key moved the robot forward in the direction it is facing.
